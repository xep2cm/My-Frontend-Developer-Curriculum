Полный учебный план фронтенд-разработчика

6 месяцев интенсивного обучения, 5 часов в день

Неделя 1: Основы HTML & CSS

  • День 1–2: Введение в HTML5, семантическая верстка, базовые теги и атрибуты.
  • День 3–4: Начало изучения CSS, селекторы, правила каскадности, позиционирование.
  • День 5–6: Работа с сеточной моделью (Flexbox и Grid Layout), медиазапросы.
  • День 7: Практический кейс: сверстать адаптивную страницу-лендинг с реализацией всех пройденных техник.

Неделя 2: Улучшение CSS и начало работы с JavaScript

  • День 1–2: Продвинутые техники CSS (анимации, переходы, трансформации).
  • День 3–4: Начинаем изучение JavaScript: синтаксис, переменные, условные конструкции, циклы.
  • День 5–6: Функции в JavaScript, области видимости, анонимные и стрелочные функции.
  • День 7: Практический кейс: реализация динамического элемента на странице с использованием JS.

Неделя 3: Работа с DOM и событиями

  • День 1–2: Манипуляции с DOM: выбор элементов, изменение свойств и стиля.
  • День 3–4: Работа с событиями: обработка кликов, клавиатуры, изменения размеров окна.
  • День 5–6: Асинхронность: Promises, async/await, fetch API.
  • День 7: Практический кейс: сделать интерактивную галерею с изменениями по событию клика.

Неделя 4: Модульность и работа с API

  • День 1–2: ES-модули, импорт и экспорт, упаковка пакетов.
  • День 3–4: Работа с API, отправка HTTP-запросов, получение и обработка данных.
  • День 5–6: Локальное хранение данных: Local Storage, Cookies.
  • День 7: Практический кейс: разработать небольшую веб-форму, принимающую вводимые данные и записывающую их в хранилище.

Неделя 5: Изучение React.js

  • День 1–2: Первое знакомство с React: компоненты, state, props.
  • День 3–4: Хуки в React: useState, useEffect, useReducer.
  • День 5–6: Роутинг в React: установка и настройка react-router-dom.
  • День 7: Практический кейс: построить простой сайт-каталог товаров с фильтрами и пагинацией.

Неделя 6: Продвинутый React и работа с состоянием

  • День 1–2: Контекст в React, context API.
  • День 3–4: Библиотека Redux для сложного управления состоянием.
  • День 5–6: Авторизация и аутентификация в приложениях React.
  • День 7: Практический кейс: создать React-приложение с авторизацией и управлением профилем пользователя.

Неделя 7: Адаптация и архитектура React-приложений

  • День 1–2: Адаптивность в React: responsive design, адаптивные компоненты.
  • День 3–4: Продуктивное использование HOC и Render Props.
  • День 5–6: Производительность React-приложений: мемоизация, виртуализации списков.
  • День 7: Практический кейс: спроектировать и реализовать сложный React-компонент с высокой производительностью.

Неделя 8: Второй фреймворк: Vue.js или Angular

  • День 1–2: Основы второго фреймворка: принципы работы, особенности архитектуры.
  • День 3–4: Сравнение двух выбранных фреймворков, выявление преимуществ и недостатков.
  • День 5–6: Повторение аналогичных концепций из React во втором фреймворке.
  • День 7: Практический кейс: повторение одной из предыдущих задач с использованием нового фреймворка.

Неделя 9: Работа с API и библиотеками

  • День 1–2: Практикум работы с популярными библиотеками (Axios, Swiper, lodash).
  • День 3–4: Глубокая интеграция внешних API-сервисов (OpenWeatherMap, Spotify API).
  • День 5–6: Отладка и профилирование приложений, best practices в работе с библиотеками.
  • День 7: Практический кейс: разработать клиентское приложение с глубокой интеграцией внешнего API.

Неделя 10: Продвинутые инструменты и лучшие практики

  • День 1–2: TypeScript: введение, преимущества, первая программа.
  • День 3–4: Безопасность в фронтенде: XSS, CSRF защита, SSL/TLS.
  • День 5–6: Работа с системами контроля версий (Git, GitHub, GitLab).
  • День 7: Практический кейс: переделать одно из существующих приложений с внедрением TypeScript и защиты безопасности.

Неделя 11: Оптимизация производительности и сборка проектов

  • День 1–2: Минимизация ресурсов, Webpack и Babel.
  • День 3–4: Lazy Loading, Preloading, Code Splitting.
  • День 5–6: Build automation tools: Grunt, Gulp, Parcel.
  • День 7: Практический кейс: оптимизировать производительность существующего проекта.

Неделя 12: Готовимся к карьере

  • День 1–2: Портфолио Development: создание персонального портфолио с демонстрацией выполненных работ.
  • День 3–4: CV Writing: правильное составление резюме и сопроводительного письма.
  • День 5–6: Job Interview Preparation: подготовка к техническому собеседованию.
  • День 7: Mock interview session: практика прохождения технического интервью.

Неделя 13: Архитектурные паттерны и масштабируемость

  • День 1–2: Микрофронтенды и модульная структура приложения.
  • День 3–4: Шаблоны проектирования UI: Flux, MVC, MVVM.
  • День 5–6: Компоненты высшего порядка (HOCs), Render Props, Hooks.
  • День 7: Практика: рефакторинг крупного компонента с применением архитектурных подходов.

Неделя 14: Интеграция и взаимодействие с бэкендом

  • День 1–2: GraphQL: основы и применение.
  • День 3–4: Создание собственного GraphQL-сервера.
  • День 5–6: Использование GraphQL в проекте с Apollo Client или Relay.
  • День 7: Практическое задание: интеграция реального бэкенда с фронтендом через GraphQL.

Неделя 15: Инструменты CI/CD и DevOps

  • День 1–2: Автоматизированное тестирование (Jest, Cypress).
  • День 3–4: Непрерывная интеграция и доставка (CircleCI, Jenkins, Travis CI).
  • День 5–6: Развертывание фронтенд-приложений (Netlify, Vercel, AWS Amplify).
  • День 7: Проект: настроить CI/CD pipeline для одного из созданных ранее проектов.

Неделя 16: Современные тенденции и проектирование интерфейсов

  • День 1–2: Доступность интерфейса (Accessibility): WCAG стандарты, ARIA роли.
  • День 3–4: Материал-дизайн и современные дизайн-системы.
  • День 5–6: Темная тема (Dark Mode): разработка и поддержка.
  • День 7: Реализация проекта с дизайном, доступным для пользователей с ограниченными возможностями и поддержкой Dark Mode.

Закрепление всего пройденного материала путём самостоятельного завершения одного комплексного проекта, подобного порталу объявлений, сайту электронной коммерции или социальной сети. Оценивание производится по критериям качества кода, адаптивности, эффективности и соблюдения лучших практик.
